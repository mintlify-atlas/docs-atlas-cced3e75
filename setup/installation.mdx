## Overview

Miniswimmer is a Next.js 15 SaaS application for swimming schools built with React 19, TypeScript, Tailwind CSS, and Prisma. This guide will walk you through the complete installation process.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js**: Version 14.0.0 or higher
- **npm** or **yarn**: Package manager
- **PostgreSQL**: Database server
- **Git**: Version control

<Note>
  The platform requires Node.js 14.0.0 or higher as specified in the package.json engines field.
</Note>

## Installation Steps

<Steps>
  <Step title="Clone the Repository">
    Clone the Miniswimmer repository to your local machine:

    ```bash
    git clone <repository-url>
    cd miniswimmer-public-site
    ```
  </Step>

  <Step title="Install Dependencies">
    Install all required npm packages:

    ```bash
    npm install
    ```

    This will install all dependencies including:
    - **Next.js 15.2.8** - React framework
    - **React 19.0.0** - UI library
    - **TypeScript 5.3.3** - Type safety
    - **Tailwind CSS 3.4.4** - Styling
    - **Prisma** - Database ORM
    - **next-intl 3.26.5** - Internationalization
    - **EmailJS 4.4.1** - Email service
    - **React Hook Form 7.56.2** - Form management
  </Step>

  <Step title="Configure Environment Variables">
    Create a `.env` file in the root directory and configure all required environment variables.

    <Warning>
      Do not commit your `.env` file to version control. Add it to `.gitignore`.
    </Warning>

    See the [Environment Variables](/setup/environment-variables) page for complete configuration details.
  </Step>

  <Step title="Set Up the Database">
    Initialize your PostgreSQL database and run Prisma migrations:

    ```bash
    # Generate Prisma Client
    npx prisma generate

    # Run database migrations
    npx prisma migrate dev

    # (Optional) Open Prisma Studio to view your database
    npx prisma studio
    ```

    <Tip>
      Prisma Studio provides a visual interface to view and edit your database at `http://localhost:5555`
    </Tip>
  </Step>

  <Step title="Configure Integrations">
    Update the `integrations.config.tsx` file to enable/disable platform features:

    ```tsx integrations.config.tsx
    const integrations = {
      isSanityEnabled: false,
      isOpenAIEnabled: false,
      isAlgoliaEnabled: false,
      isMailchimpEnabled: false,
      isAuthEnabled: true,
      isPaymentsEnabled: false,
      isI18nEnabled: true,
    };
    ```

    See the [Configuration](/setup/configuration) page for detailed integration setup.
  </Step>

  <Step title="Run Development Server">
    Start the development server:

    ```bash
    npm run dev
    ```

    The application will be available at `http://localhost:3000`
  </Step>
</Steps>

## Package Scripts

The platform includes several npm scripts for different tasks:

### Development

```bash
# Start development server
npm run dev

# Start production server
npm run start
```

### Building

```bash
# Build for production
npm run build

# Test build with linting and formatting checks
npm run test-build
```

<Note>
  The `postbuild` script automatically generates a sitemap after each build using `scripts/generate-sitemap.js`
</Note>

### Code Quality

```bash
# Check code formatting
npm run check-format

# Check linting errors
npm run check-lint

# Check both formatting and linting
npm run check-style

# Fix formatting issues
npm run fix-format

# Fix linting issues
npm run fix-lint

# Fix all style issues
npm run fix-style
```

### Stripe Webhooks (Development)

```bash
# Listen for Stripe webhook events locally
npm run stripe:listen
```

<Warning>
  Ensure you have the Stripe CLI installed and configured before running webhook listeners.
</Warning>

## Verification

After installation, verify everything is working:

<Steps>
  <Step title="Check Development Server">
    Visit `http://localhost:3000` and ensure the homepage loads correctly.
  </Step>

  <Step title="Verify Database Connection">
    Check that Prisma can connect to your database:

    ```bash
    npx prisma db pull
    ```
  </Step>

  <Step title="Test Internationalization">
    Switch between English and Spanish to verify i18n is working.
  </Step>

  <Step title="Run Type Checking">
    Ensure there are no TypeScript errors:

    ```bash
    npx tsc --noEmit
    ```
  </Step>
</Steps>

## Troubleshooting

<Accordion title="Node version mismatch">
  If you encounter Node version issues, ensure you're using Node.js 14.0.0 or higher:

  ```bash
  node --version
  ```

  Consider using [nvm](https://github.com/nvm-sh/nvm) to manage Node versions.
</Accordion>

<Accordion title="Database connection errors">
  Verify your `DATABASE_URL` in the `.env` file is correct and PostgreSQL is running:

  ```bash
  # Check PostgreSQL status
  pg_isready
  ```
</Accordion>

<Accordion title="Prisma Client not generated">
  If you get Prisma Client errors, regenerate it:

  ```bash
  npx prisma generate
  ```
</Accordion>

<Accordion title="Port 3000 already in use">
  If port 3000 is occupied, specify a different port:

  ```bash
  PORT=3001 npm run dev
  ```
</Accordion>
