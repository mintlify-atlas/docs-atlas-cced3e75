Complete implementation at `src/components/Common/ReagendamientoForm/index.tsx:139`:

```typescript
const dateSlots = useMemo(() => {
  const now = new Date();
  const startDate = new Date(now.getTime() + 12 * 60 * 60 * 1000);
  const endDate = new Date(now.getTime() + 21 * 24 * 60 * 60 * 1000);

  const dates: { value: string; label: string; weekday: string }[] = [];
  let current = new Date(startDate);
  current.setHours(0, 0, 0, 0);

  const todayKey = now.toISOString().split('T')[0];
  if (current.toISOString().split('T')[0] === todayKey) {
    current.setDate(current.getDate() + 1);
  }

  const end = new Date(endDate);
  end.setHours(23, 59, 59, 999);

  while (current <= end) {
    const dateKey = current.toISOString().split('T')[0];
    const weekday = current.toLocaleDateString('es-CL', { weekday: 'long' });
    const label = current.toLocaleDateString('es-CL', {
      day: 'numeric',
      month: 'short',
    });
    dates.push({ value: dateKey, label, weekday });
    current.setDate(current.getDate() + 1);
  }

  return dates;
}, []);
```

## Key Features

<Note>
  **12-Hour Minimum Notice**: Students must request rescheduling at least 12 hours before their class, or the request will be invalid.

- **Multi-location support**: Works across all Miniswimmer locations

- **Email notifications**: Staff receives detailed rescheduling requests via EmailJS
- **Form validation**: Client-side validation prevents invalid submissions
- **Loading states**: Visual feedback during form submission
- **Success/error handling**: Clear messaging for all submission outcomes
- **Responsive design**: Works seamlessly on mobile, tablet, and desktop

## Best Practices

<Tip>
  Always test the form with various edge cases, such as attempting to book on the same day or selecting dates beyond the 21-day window.
</Tip>

- **Validate on client and server**: Never trust client-side validation alone
- **Provide clear error messages**: Help users understand what went wrong
- **Test email delivery**: Ensure EmailJS credentials are correct
- **Monitor submission rate**: Implement rate limiting to prevent abuse
- **Log failed submissions**: Track issues for debugging
