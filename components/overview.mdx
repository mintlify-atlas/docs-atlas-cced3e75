## Introduction

The Miniswimmer platform is built with **Next.js 15** and uses a modular component architecture that emphasizes reusability, internationalization, and type safety. All components are written in TypeScript and follow React best practices.

## Architecture Principles

### Component Organization

Components are organized into three main categories:

1. **Layout Components** - Header, Footer, and structural elements
2. **Common Components** - Reusable UI components like Hero, Features, Testimonials
3. **Form Components** - Contact forms, test class booking forms with validation

### Key Technologies

<Note>
  The platform leverages modern React patterns including hooks, context, and server/client component separation.
</Note>

- **Next.js 15** - React framework with App Router
- **TypeScript** - Type-safe component development
- **next-intl** - Internationalization (i18n) support
- **React Hook Form** - Form state management
- **Yup** - Schema validation
- **EmailJS** - Email service integration
- **Tailwind CSS** - Utility-first styling

## Component Patterns

### Internationalization Pattern

All components use the `next-intl` library for translations:

```tsx
import { useTranslations } from 'next-intl';

const Hero = (props: any) => {
  const { pageTraslation } = props;
  const t = useTranslations(`${pageTraslation}.hero_section`);
  
  return (
    <h1>{t('title')}</h1>
  );
};
```

<Tip>
  Translation keys follow the pattern: `{page}.{section}.{key}` for organized locale management.
</Tip>

### Props Interface Pattern

Most components accept a `pageTraslation` prop to specify which translation namespace to use:

```tsx
interface ComponentProps {
  pageTraslation: string;
}

const MyComponent = ({ pageTraslation }: ComponentProps) => {
  // Component logic
};
```

### Client/Server Component Separation

Components that use hooks or browser APIs are marked with `'use client'`:

```tsx
'use client';

import { useState } from 'react';

const InteractiveComponent = () => {
  const [state, setState] = useState();
  // ...
};
```

## Styling Conventions

### Tailwind CSS

All components use Tailwind CSS utility classes:

- **Responsive Design**: Uses `sm:`, `lg:`, `xl:` breakpoints
- **Dark Mode**: Supports dark mode with `dark:` prefix
- **Custom Colors**: Primary color scheme defined in Tailwind config

```tsx
<div className="mx-auto w-full max-w-[1170px] px-4 sm:px-8 xl:px-0">
  <h1 className="font-satoshi text-heading-4 font-bold text-black dark:text-white">
    Title
  </h1>
</div>
```

### Typography

The platform uses the **Satoshi** font family:

- `font-satoshi` - Base font class
- Font weights: `font-medium`, `font-bold`
- Custom tracking: `-tracking-[1.6px]`

## State Management

### Local State

Components manage local state using React hooks:

- `useState` - Component state
- `useEffect` - Side effects and lifecycle
- `useRef` - DOM references and mutable values

### Form State

Forms use React Hook Form for state management:

```tsx
const { register, handleSubmit, formState: { errors } } = useForm<FormData>({
  resolver: yupResolver(schema)
});
```

## Type Safety

### TypeScript Interfaces

Components use TypeScript interfaces for props and data structures:

```tsx
interface FeatureWithImg {
  title: string;
  subtitle: string;
  subtitle2: string;
  img: string;
  features: string[];
}
```

### Type Definitions

Common types are stored in `src/types/`:

- `menu.ts` - Menu structure types
- `testimonial.ts` - Testimonial data types
- `featureWithImg.ts` - Feature component types

## Performance Optimization

<Note>
  Components use Next.js Image component for automatic image optimization.
</Note>

### Image Optimization

```tsx
import Image from 'next/image';

<Image
  src="/images/hero.png"
  alt="Hero Image"
  width={600}
  height={500}
  priority // For above-fold images
/>
```

### Code Splitting

Next.js automatically code-splits components for optimal loading performance.

## Component Communication

### Props Drilling

Parent components pass data down through props:

```tsx
<Hero pageTraslation="homepage" />
<Features pageTraslation="homepage" />
```

### Callbacks

Child components communicate with parents through callback props:

```tsx
<Dropdown setNavbarOpen={setNavbarOpen} />
```

## Best Practices

1. **Component Reusability** - Build components that can be used across multiple pages
2. **Type Safety** - Always define TypeScript interfaces for props and data
3. **Accessibility** - Use semantic HTML and ARIA labels
4. **Internationalization** - Support multiple languages from the start
5. **Responsive Design** - Mobile-first approach with Tailwind breakpoints
6. **Performance** - Lazy load components when appropriate
7. **Error Handling** - Implement proper error boundaries and validation
