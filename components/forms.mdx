## Overview

Form components in the Miniswimmer platform use **React Hook Form** for state management and **Yup** for schema validation. Forms are integrated with **EmailJS** for email delivery.

## ContactForm Component

Simple contact form with name, email, and message fields (`src/components/Common/ContactForm/index.tsx`).

### Features

- Form validation with Yup schema
- Real-time error messages
- EmailJS integration
- Success/failure feedback
- Loading states
- Form reset on success

### Usage

```tsx
import ContactForm from '@/components/Common/ContactForm';

<ContactForm pageTraslation="contacto" />
```

### Implementation

<CodeGroup>
```tsx ContactForm Component
'use client';

import { useState } from 'react';
import Link from 'next/link';
import { useTranslations } from 'next-intl';
import { useForm } from 'react-hook-form';
import { yupResolver } from '@hookform/resolvers/yup';
import * as yup from 'yup';
import emailjs from '@emailjs/browser';

const schema = yup.object({
  nombre: yup.string().required('El nombre es requerido'),
  email: yup.string().email('Email inválido').required('El email es requerido'),
  mensaje: yup.string().required('El mensaje es requerido')
}).required();

type FormData = yup.InferType<typeof schema>;

const Features = (props: any) => {
  const [isSentEmail, setIsSentEmail] = useState({
    sentEmail: false,
    isFailure: false,
    title: "Página no encontrada 😭",
    text: "Parece que no podemos encontrar la página que estás buscando.",
  });
  
  const [isSubmitting, setIsSubmitting] = useState(false);
  
  const {
    register,
    handleSubmit,
    formState: { errors },
    reset
  } = useForm<FormData>({
    resolver: yupResolver(schema)
  });

  const onSubmit = async (data: FormData) => {
    setIsSubmitting(true);
    setIsSentEmail({
      sentEmail: true,
      isFailure: false,
      title: "Conectando al satélite, por favor espere un momento ⌛",
      text: "Estamos enviando su solicitud.",
    });
    
    try {
      const templateParams = {
        from_name: data.nombre,
        from_email: data.email,
        message: data.mensaje,
        to_name: 'Miniswimmer',
      };

      await emailjs.send(
        'service_ucb8wga',
        'template_3z2nv7b',
        templateParams,
        'Csc41asZklkk5HTWk'
      );

      reset();
      setIsSentEmail({
        sentEmail: true,
        isFailure: false,
        title: "Mensaje enviado 🎉",
        text: "Nos pondremos en contacto con usted lo antes posible.",
      });
    } catch (error) {
      console.error('Error al enviar el email:', error);
      setIsSentEmail({
        sentEmail: true,
        isFailure: true,
        title: "No se pudo enviar el mensaje 😭",
        text: "Puedes enviarnos un mensaje a welcome@miniswimmer.cl.",
      });
    } finally {
      setIsSubmitting(false);
    }
  };

  return (
    <section className='relative z-1 overflow-hidden bg-[linear-gradient(90deg,#7e7e7e_0%,#3b3b3b_100%)] py-17.5'>
      <div className='relative z-1 mx-auto w-full max-w-[1170px] px-4 sm:px-8 xl:px-0'>
        <div className='flex justify-center items-center'>
          <div className='max-w-[600px] w-full bg-white border rounded-[24px] p-[48px]'>
            {isSentEmail.sentEmail ? (
              {/* Success/Error Message */}
            ) : (
              <form onSubmit={handleSubmit(onSubmit)} className="space-y-4 w-full">
                <div>
                  <label htmlFor="nombre" className="block text-lg font-light text-gray-700">
                    Nombre
                  </label>
                  <input
                    {...register('nombre')}
                    type="text"
                    id="nombre"
                    className="mt-1 mb-12 block w-full h-[48px] border rounded-[6px] p-[13px]"
                  />
                  {errors.nombre && (
                    <p className="-mt-10 text-sm text-red-300">{errors.nombre.message}</p>
                  )}
                </div>

                <div>
                  <label htmlFor="email" className="block text-lg font-light text-gray-700">
                    Email
                  </label>
                  <input
                    {...register('email')}
                    type="email"
                    id="email"
                    className="mt-1 mb-12 block w-full h-[48px] border rounded-[6px] p-[13px]"
                  />
                  {errors.email && (
                    <p className="-mt-10 text-sm text-red-300">{errors.email.message}</p>
                  )}
                </div>

                <div>
                  <label htmlFor="mensaje" className="block text-lg font-light text-gray-700">
                    ¿Cómo podemos ayudarle?
                  </label>
                  <textarea
                    {...register('mensaje')}
                    id="mensaje"
                    rows={4}
                    className="mt-1 mb-12 block w-full border rounded-[6px] p-[13px]"
                  />
                  {errors.mensaje && (
                    <p className="-mt-10 text-sm text-red-300">{errors.mensaje.message}</p>
                  )}
                </div>

                <button
                  type="submit"
                  disabled={isSubmitting}
                  className="w-full rounded-md bg-primary px-4 py-2 text-white hover:bg-primary/95 disabled:opacity-50"
                >
                  {isSubmitting ? 'Enviando...' : 'Enviar'}
                </button>
              </form>
            )}
          </div>
        </div>
      </div>
    </section>
  );
};
```

### Date Slot Generation

<CodeGroup>
```typescript Configuration
const startDate = "25-09-2025";
const endDate = "01-12-2025";
const datesWithoutSlot = [
  "18-09-2025",
  "19-09-2025",
  "12-10-2025",
  "31-10-2025",
  "01-11-2025",
  "16-11-2025",
  "08-12-2025",
  "14-12-2025",
  "25-12-2025",
  "01-01-2026",
];

const timeSlots = [
  { day: "jueves", hour: "13:00", service: "Natación embarazadas" },
  { day: "jueves", hour: "14:00", service: "Natación bebes y niños" },
  { day: "jueves", hour: "14:30", service: "Natación bebes y niños" },
  { day: "jueves", hour: "15:00", service: "Natación bebes y niños" },
  { day: "jueves", hour: "15:30", service: "Natación bebes y niños" }
];
```

```typescript Slot Generation Function
interface AvailableDate {
  id: string;
  date: string;
  hour: string;
  service: string;
}

const generateAvailableDates = (
  startDateStr: string, 
  endDateStr: string
): AvailableDate[] => {
  const today = new Date();
  today.setHours(0, 0, 0, 0);
  
  const [startDay, startMonth, startYear] = startDateStr.split('-').map(Number);
  const startDateObj = new Date(startYear, startMonth - 1, startDay);
  
  const [endDay, endMonth, endYear] = endDateStr.split('-').map(Number);
  const endDateObj = new Date(endYear, endMonth - 1, endDay);
  
  const blockedDates = datesWithoutSlot.map(dateStr => {
    const [day, month, year] = dateStr.split('-').map(Number);
    const blockedDate = new Date(year, month - 1, day);
    blockedDate.setHours(0, 0, 0, 0);
    return blockedDate;
  });
  
  const availableDates: AvailableDate[] = [];
  let currentDate = new Date(startDateObj);
  
  while (currentDate <= endDateObj) {
    // Only generate slots for Thursday (day 4)
    if (currentDate.getDay() === 4) {
      const isBlocked = blockedDates.some(blockedDate => 
        blockedDate.getTime() === currentDate.getTime()
      );
      
      if (!isBlocked) {
        timeSlots.forEach(slot => {
          const slotId = `${currentDate.getTime()}-${slot.hour.replace(':', '')}`;
          
          availableDates.push({
            id: slotId,
            date: currentDate.toLocaleDateString('es-ES', { 
              day: 'numeric', 
              month: 'long' 
            }),
            hour: slot.hour,
            service: slot.service
          });
        });
      }
    }
    
    currentDate.setDate(currentDate.getDate() + 1);
  }
  
  return availableDates;
};
```

## Best Practices

<Note>
  Always validate forms on both client and server (if applicable) for security.
</Note>

1. **Use TypeScript** - Define form data types with Yup inference
2. **Schema Validation** - Centralize validation rules in Yup schemas
3. **Error Handling** - Display user-friendly error messages
4. **Loading States** - Show feedback during async operations
5. **Form Reset** - Clear form after successful submission
6. **Accessibility** - Use proper labels and ARIA attributes
7. **Responsive Design** - Test on all device sizes
8. **Email Templates** - Configure EmailJS templates for consistent emails

## Yup Validation Patterns

### Common Validations

```typescript
// Required field
yup.string().required('Field is required')

// Email validation
yup.string().email('Invalid email').required('Email is required')

// Min/Max length
yup.string().min(3, 'Min 3 characters').max(50, 'Max 50 characters')

// Phone number (custom regex)
yup.string().matches(/^[0-9]+$/, 'Must be only digits')

// Conditional validation
yup.string().when('otherField', {
  is: 'value',
  then: yup.string().required('Required when otherField is value')
})
```

## Troubleshooting

### Common Issues

<Tabs>
  <Tab title="Validation Not Working">
    - Ensure `yupResolver` is imported from `@hookform/resolvers/yup`
    - Check that schema is passed to `useForm` resolver
    - Verify field names match between schema and `register()`
  </Tab>
  
  <Tab title="EmailJS Errors">
    - Verify Service ID, Template ID, and Public Key are correct
    - Check EmailJS dashboard for service status
    - Ensure template variables match parameter names
    - Check browser console for specific error messages
  </Tab>
  
  <Tab title="Form Not Resetting">
    - Call `reset()` after successful submission
    - Ensure `reset` is destructured from `useForm()`
    - Check that form state is properly managed
  </Tab>
</Tabs>
