## Overview

Miniswimmer uses EmailJS (@emailjs/browser v4.4.1) to send emails directly from the client-side without requiring a backend server. This integration powers all contact forms, test class registrations, and rescheduling requests.

## Installation

EmailJS is already included in the project dependencies:

```json package.json
{
  "dependencies": {
    "@emailjs/browser": "^4.4.1"
  }
}
```

To install or update:

```bash
npm install @emailjs/browser
```

## Configuration

### Environment Setup

EmailJS requires three configuration values:

1. **Service ID**: Your email service provider (e.g., Gmail, Outlook)
2. **Template ID**: The email template to use
3. **Public Key**: Your EmailJS account public key

<Note>
  Currently, these values are hardcoded in the components. For production, consider moving them to environment variables:
  - `NEXT_PUBLIC_EMAILJS_SERVICE_ID`
  - `NEXT_PUBLIC_EMAILJS_PUBLIC_KEY`
</Note>

## Implementation

### Basic Contact Form

Here's the complete implementation from `/src/components/Common/ContactForm/index.tsx`:

```tsx ContactForm.tsx
'use client';

import { useState } from 'react';
import { useForm } from 'react-hook-form';
import { yupResolver } from '@hookform/resolvers/yup';
import * as yup from 'yup';
import emailjs from '@emailjs/browser';

// Form validation schema
const schema = yup.object({
  nombre: yup.string().required('El nombre es requerido'),
  email: yup.string().email('Email inválido').required('El email es requerido'),
  mensaje: yup.string().required('El mensaje es requerido')
}).required();

type FormData = yup.InferType<typeof schema>;

function ContactForm() {
  const [isSubmitting, setIsSubmitting] = useState(false);
  const [isSentEmail, setIsSentEmail] = useState({
    sentEmail: false,
    isFailure: false,
    title: "",
    text: "",
  });
  
  const {
    register,
    handleSubmit,
    formState: { errors },
    reset
  } = useForm<FormData>({
    resolver: yupResolver(schema)
  });

  const onSubmit = async (data: FormData) => {
    setIsSubmitting(true);
    setIsSentEmail({
      sentEmail: true,
      isFailure: false,
      title: "Conectando al satélite, por favor espere un momento ⌛",
      text: "Estamos enviando su solicitud.",
    });
    
    try {
      const templateParams = {
        from_name: data.nombre,
        from_email: data.email,
        message: data.mensaje,
        to_name: 'Miniswimmer',
      };

      await emailjs.send(
        'service_ucb8wga',
        'template_3z2nv7b',
        templateParams,
        'Csc41asZklkk5HTWk'
      );

      reset();
      setIsSentEmail({
        sentEmail: true,
        isFailure: false,
        title: "Mensaje enviado 🎉",
        text: "Nos pondremos en contacto con usted lo antes posible.",
      });
    } catch (error) {
      console.error('Error al enviar el email:', error);
      setIsSentEmail({
        sentEmail: true,
        isFailure: true,
        title: "No se pudo enviar el mensaje 😭",
        text: "Puedes enviarnos un mensaje a welcome@miniswimmer.cl.",
      });
    } finally {
      setIsSubmitting(false);
    }
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      {/* Form fields */}
    </form>
  );
}
```

### Template Parameters

EmailJS templates receive parameters that map to your email template variables:

```tsx
const templateParams = {
  from_name: data.nombre,      // Sender's name
  from_email: data.email,      // Sender's email
  message: data.mensaje,       // Message content
  to_name: 'Miniswimmer',     // Recipient name
};
```

<Tip>
  Template parameter names must exactly match the variable names defined in your EmailJS template.
</Tip>

## Best Practices

### 1. Client-Side Only

EmailJS must run in client components:

```tsx
'use client';

import emailjs from '@emailjs/browser';
```

### 2. Error Handling

Always implement comprehensive error handling:

```tsx
try {
  await emailjs.send(serviceId, templateId, params, publicKey);
  // Success handling
} catch (error) {
  console.error('Error al enviar el email:', error);
  // Error handling with fallback contact info
}
```

### 3. Rate Limiting

<Warning>
  EmailJS has rate limits based on your plan. Implement client-side throttling to prevent abuse:
  - Disable submit button during sending
  - Add cooldown period between submissions
  - Consider adding CAPTCHA for public forms
</Warning>

### 4. Form Reset

Always reset the form after successful submission:

```tsx
await emailjs.send(...);
reset(); // Clear all form fields
```

### 5. Loading States

Prevent double submissions by disabling the button:

```tsx
<button
  type="submit"
  disabled={isSubmitting}
  className="disabled:opacity-50"
>
  {isSubmitting ? 'Enviando...' : 'Enviar'}
</button>
```

## Testing

### Test Mode

EmailJS doesn't have a dedicated test mode, but you can:

1. Create a separate test template
2. Send emails to a test address
3. Use conditional logic based on environment:

```tsx
const serviceId = process.env.NODE_ENV === 'production' 
  ? 'service_ucb8wga' 
  : 'service_test';
```

### Console Logging

Log template parameters during development:

```tsx
if (process.env.NODE_ENV === 'development') {
  console.log('EmailJS params:', templateParams);
}
```

## Troubleshooting

### Common Errors

**"Failed to send email"**
- Verify service ID, template ID, and public key
- Check EmailJS dashboard for account status
- Ensure template variables match parameter names

**"Template not found"**
- Confirm template ID exists in your EmailJS account
- Check that the template is published

**"Rate limit exceeded"**
- Upgrade your EmailJS plan
- Implement client-side rate limiting
- Add CAPTCHA protection

### Network Issues

<Note>
  EmailJS requires internet connectivity. Implement proper error handling to inform users if the submission fails due to network issues.
</Note>
